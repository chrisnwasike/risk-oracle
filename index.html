How Risk Oracle Works

Simple, deterministic, transparent

1

Observe Behavior

Off-chain indexer monitors wallet transaction patterns across the chain, no personal data, just on-chain behavior.

2

Classify Deterministically

Rule engine applies explicit, auditable rules. No ML black box. Same input = same output, always.

3

Gate On-Chain

Protocols query oracle contract: "Can this wallet perform this action?" Oracle responds: yes or no.

4

Cross-Protocol Trust

Tier earned on one protocol carries over to others. Network effects create ecosystem-wide safety.

Integration in Minutes

Three lines of code. That's it.

Solidity Contract

interface IRiskOracle {
    function can(address wallet, uint8 actionType) 
        external view returns (bool);
}

contract MyProtocol {
    IRiskOracle oracle = IRiskOracle(0x5352...);
    
    function trade() external {
        // Gate risky action by tier
        require(oracle.can(msg.sender, 1), "Need Tier 2+");
        // ... your trading logic
    }
}

TypeScript SDK

import { createClient, ActionType } from '@risk-oracle/sdk';

const client = createClient();

// Check if wallet can trade
const canTrade = await client.can(
    '0x1234...', 
    ActionType.TRADE
);

if (!canTrade) {
    throw new Error('Insufficient tier');
}
